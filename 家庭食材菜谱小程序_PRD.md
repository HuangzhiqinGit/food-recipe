## 2. 产品概述

### 2.1 产品定位

| 项目 | 描述 |
|------|------|
| **产品名称** | 家庭食材&菜谱管理小程序 |
| **产品类型** | 生活工具类 |
| **平台载体** | 微信小程序 |
| **产品Slogan** | "智能管理食材，轻松搞定一日三餐" |
| **商业模式** | 免费工具 + 未来可扩展会员增值服务 |

### 2.2 核心价值主张

1. **智能食材管理**: 轻松记录食材，自动提醒临期，减少浪费
2. **一键菜谱匹配**: 根据现有食材智能推荐可做的菜
3. **便捷购物清单**: 从菜谱一键添加缺失食材，采购不遗漏
4. **图片化管理**: 食材、食谱可视化，更直观
5. **智能库存扣减**: 做菜自动扣减食材，库存实时更新

## 3. 功能模块详细设计

### 3.1 首页模块

#### 3.1.1 今日推荐菜谱

**用户故事**:
> 作为用户，我希望首页能看到今日推荐的菜谱，以便快速决定今天吃什么。

**功能描述**:
- 首页新增"今日推荐"栏位
- 根据用户历史偏好、季节、时段智能推荐
- 显示菜谱图片、名称、烹饪时长、难度
- 点击可查看详情或开始烹饪

**推荐算法**:
1. 优先推荐用户收藏的菜谱
2. 根据用户现有食材匹配度排序
3. 考虑烹饪时长（工作日推荐快手菜）
4. 考虑季节因素

**界面元素**:
```
┌─────────────────────────────────────┐
│ 今日推荐                            │
│ ┌─────────┐ ┌─────────┐ ┌─────────┐│
│ │ 图片    │ │ 图片    │ │ 图片    ││
│ │ 红烧肉  │ │ 番茄炒蛋│ │ 清蒸鱼  ││
│ │ ⏱30分钟│ │ ⏱15分钟│ │ ⏱25分钟││
│ └─────────┘ └─────────┘ └─────────┘│
└─────────────────────────────────────┘
```

### 3.2 食材模块

#### 3.2.1 食材图片上传

**用户故事**:
> 作为用户，我希望给食材拍照上传图片，以便更直观地识别和管理食材。

**功能描述**:
- 添加/编辑食材时支持图片上传
- 支持拍照上传
- 支持从相册选择
- 图片压缩后上传（最大2MB）
- 图片存储到阿里云OSS

**交互流程**:
```
点击添加食材
    ↓
填写食材名称、数量等信息
    ↓
点击"上传图片"按钮
    ↓
弹出选择：拍照 / 从相册选择
    ↓
选择/拍摄图片
    ↓
图片预览，确认上传
    ↓
保存食材
```

**界面元素**:
```
┌─────────────────────────────────────┐
│ 食材图片                            │
│    ┌─────────────┐                  │
│    │             │                  │
│    │   + 点击    │                  │
│    │   上传图片  │                  │
│    │             │                  │
│    └─────────────┘                  │
│                                     │
│ [拍照] [从相册选择]                 │
└─────────────────────────────────────┘
```

#### 3.2.2 扫描录入食材

**用户故事**:
> 作为用户，我希望通过扫描食材包装自动识别信息，以便快速录入食材。

**功能描述**:
- 支持扫描食材包装上的条形码/二维码
- OCR识别包装上的文字信息（名称、保质期等）
- 自动填充食材信息
- 支持手动修正识别结果

**技术方案**:
- 调用微信扫码API
- 后端对接商品条码数据库
- OCR使用阿里云/腾讯云文字识别服务

**交互流程**:
```
点击"扫描录入"
    ↓
调用相机扫码
    ↓
识别条码/包装文字
    ↓
自动填充食材信息
    ↓
用户确认/修改
    ↓
保存食材
```

**用户故事**:
> 作为用户，我希望在食材列表中看到更多信息，以便更好地管理食材。

#### 3.2.4 临期/过期食材筛选

**用户故事**:
> 作为用户，我希望快速筛选出临期或过期的食材，以便及时处理。

**功能描述**:
- 新增筛选条件：全部 / 新鲜 / 临期 / 已过期
- 临期定义：3天内过期
- 已过期定义：已过保质期
- 筛选结果按过期时间排序

**筛选选项**:
```
[全部] [新鲜] [临期] [已过期]
```

**状态颜色**:
| 状态 | 颜色 | 说明 |
|-----|------|------|
| 新鲜 | 绿色 | 过期时间 > 3天 |
| 临期 | 橙色 | 0 < 过期时间 ≤ 3天 |
| 已过期 | 红色 | 过期时间 ≤ 0 |

### 3.3 食谱模块

#### 3.3.1 食谱图片上传

#### 3.3.2 选择食谱自动扣减库存

**用户故事**:
> 作为用户，我希望选择食谱做菜后自动扣减食材库存，以便实时掌握食材数量。

**功能描述**:
- 食谱详情页增加"开始烹饪"按钮
- 点击后显示所需食材清单
- 确认后自动扣减对应食材库存
- 库存不足时提示补充

**交互流程**:
```
浏览食谱
    ↓
点击"开始烹饪"
    ↓
显示所需食材清单
    ↓
检查库存是否充足
    ↓
确认烹饪
    ↓
自动扣减食材库存
    ↓
提示烹饪开始
```

**扣减规则**:
- 按食谱所需数量扣减
- 库存不足时标记为"需购买"
- 支持手动调整扣减数量

## 5. 接口设计

### 5.1 新增接口

#### 5.1.1 图片上传

```
POST /api/v1/upload/image
Content-Type: multipart/form-data

Request:
  - file: 图片文件
  - type: 上传类型（food/recipe）

Response:
  {
    "code": 200,
    "message": "success",
    "data": {
      "url": "https://xxx.aliyuncs.com/xxx.jpg"
    }
  }
```

#### 5.1.2 扫描识别

```
POST /api/v1/food/scan
Content-Type: application/json

Request:
  {
    "image": "base64编码的图片"
  }

Response:
  {
    "code": 200,
    "message": "success",
    "data": {
      "name": "西红柿",
      "brand": "xxx",
      "expireDate": "2026-06-01"
    }
  }
```

#### 5.1.3 一键入库

```
POST /api/v1/shopping/batch-stock
Content-Type: application/json

Request:
  {
    "itemIds": [1, 2, 3],
    "storageLocation": "refrigerated",
    "expireDate": "2026-04-01"
  }

Response:
  {
    "code": 200,
    "message": "success",
    "data": {
      "successCount": 3
    }
  }
```

#### 5.1.4 烹饪扣减库存

```
POST /api/v1/recipe/{id}/cook
Content-Type: application/json

Request:
  {
    "adjustments": {
      "1": 100,  // 食材ID: 扣减数量(g)
      "2": 2     // 食材ID: 扣减数量(个)
    }
  }

Response:
  {
    "code": 200,
    "message": "success",
    "data": {
      "deducted": [...],
      "insufficient": [...]
    }
  }
```
