📋 测试结论与建议

⚠️ 发现的问题

  • 受保护的 API 接口返回 500 错误，可能是数据库连接或查询异常
  • JWT Token 验证已通过，但服务端处理请求时出现异常
  • 需要检查 FoodService、RecipeService 等服务类的实现

✅ 成功的部分

  • 服务成功启动，Tomcat 运行在 8080 端口
  • 数据库连接正常，MySQL 服务运行正常
  • 登录接口的参数校验逻辑正常工作
  • JWT Token 生成和验证机制已配置

🔧 修复建议

  1. 检查 application.yml 中的数据库配置是否正确
  2. 查看服务器日志，定位具体的服务层异常
  3. 确认所有 Service 类是否正确注入了 Mapper
  4. 测试数据库查询语句是否能正常执行
